# FinancesApp v1.0

Консольное приложение для учета финансовых транзакций на Java.

## Описание

FinancesApp - это простое консольное приложение для управления финансовыми транзакциями. Позволяет добавлять, просматривать, искать и удалять транзакции с сохранением данных в файл.

## Функциональность

- ✅ Добавление финансовых транзакций с указанием суммы и категории
- ✅ Просмотр всех сохраненных транзакций
- ✅ Расчет общей суммы всех транзакций
- ✅ Поиск транзакций по категории (без учета регистра)
- ✅ Удаление транзакций по ID
- ✅ Автоматическое сохранение данных в CSV файл
- ✅ Загрузка данных при запуске приложения
- ✅ Обработка ошибок с понятными сообщениями

## Требования

- Java 8 или выше (рекомендуется Java 11+)
- Только стандартная библиотека Java (без внешних зависимостей)

## Компиляция и запуск

### Компиляция

```bash
# Создать директорию для скомпилированных классов
mkdir -p bin

# Скомпилировать все Java файлы
javac -d bin -encoding UTF-8 src/main/java/financesapp/**/*.java src/main/java/financesapp/*.java
```

### Запуск

```bash
# Запустить приложение
java -cp bin financesapp.Main
```

### Использование IDE

1. Откройте проект в IntelliJ IDEA, Eclipse или другой Java IDE
2. Убедитесь, что установлен JDK 8 или выше
3. Запустите класс `financesapp.Main`

### Примеры операций

**Добавление транзакции:**
- Введите сумму (может быть положительной или отрицательной): `1500.00`
- Введите категорию: `Доход`

**Просмотр транзакций:**
- Выберите пункт 2 из меню
- Все транзакции отобразятся в табличном формате

**Поиск по категории:**
- Выберите пункт 4 из меню
- Введите категорию для поиска (регистр не важен)

**Удаление транзакции:**
- Выберите пункт 5 из меню
- Просмотрите список транзакций
- Введите ID транзакции для удаления

## Формат хранения данных

Данные сохраняются в файл `data/transactions.csv` в формате CSV:

```csv
id,amount,category,date
1,1500.00,Доход,2024-01-15T10:30:00
2,-500.00,Покупка,2024-01-15T14:20:00
```

- **Кодировка**: UTF-8
- **Формат даты**: ISO 8601 (YYYY-MM-DDTHH:mm:ss)
- **Разделитель**: запятая

## Архитектура

Приложение построено на основе многослойной архитектуры (Layered Architecture):

- **Model Layer**: Класс `Transaction` - представление финансовой транзакции
- **Service Layer**: `TransactionService` - бизнес-логика приложения
- **Repository Layer**: `TransactionRepository` - работа с файловым хранилищем
- **UI Layer**: `ConsoleUI` - консольный интерфейс пользователя
- **Exception Layer**: Пользовательские исключения для обработки ошибок

### Принципы проектирования

- **SOLID**: Применение принципов SOLID для чистой архитектуры
- **Repository Pattern**: Абстракция доступа к данным
- **Service Layer Pattern**: Централизация бизнес-логики
- **Dependency Injection**: Внедрение зависимостей через конструкторы
- **Separation of Concerns**: Четкое разделение ответственности между слоями

## Обработка ошибок

Приложение обрабатывает следующие типы ошибок:

- Некорректный ввод суммы → запрос повторного ввода
- Пустая категория → запрос повторного ввода
- Несуществующий ID транзакции → информативное сообщение
- Ошибки работы с файлом → сообщение об ошибке, продолжение работы
- Некорректный формат данных в файле → пропуск некорректных записей
